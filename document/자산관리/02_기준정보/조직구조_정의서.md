# 문서명: 조직구조_정의서.md
- 작성일: 2025-05-23

---

## [1. 정의]

조직구조는 시스템 내 사용자, 자산, 결재 권한, 책임자 할당 등을  
**조직 단위(회사, 지사, 부서, 팀)**로 구분하여 관리하기 위한 기준이다.

모든 사용자 및 자산은 특정 조직에 속하며,  
조직을 기반으로 책임자 자동 매핑, 권한 분리, 통계 분석 등이 가능해진다.

---

## [2. 조직 구조 계층 예시]

```txt
회사 (Company)
 └ 지사 (Branch)
     └ 부서 (Department)
         └ 팀 (Team)
````

예:

```
HYBE
 └ 서울 본사
     └ 운영본부
         └ 자산관리팀
 └ 부산 지사
     └ 물류센터
```

---

## \[3. 기준 테이블 예시]

### `ZCM_ORG_UNIT` (조직 단위 테이블)

| 필드          | 설명                                     |
| ----------- | -------------------------------------- |
| orgId       | 조직 ID                                  |
| orgName     | 조직명                                    |
| parentOrgId | 상위 조직 ID (계층 구조)                       |
| orgType     | 조직 유형 (COMPANY / BRANCH / DEPT / TEAM) |
| isActive    | 사용 여부                                  |

---

## \[4. 조직 연계 활용 항목]

| 항목                               | 설명                                 |
| -------------------------------- | ---------------------------------- |
| 사용자 (`ZCM_USER.department`)      | 소속 부서 기준 설정                        |
| 자산 (`ZFI_ASSET_MASTER.location`) | 자산 소유 조직 기준 설정                     |
| 결재선 자동 지정                        | 부서 기준 결재자 자동 매핑                    |
| 권한 범위 제어                         | 조직별 Role 적용 가능                     |
| 통계/모니터링                          | 조직별 자산 등록 수, 이동 이력, 리폼/수선 건수 집계 가능 |

---

## \[5. 관리 기준]

* 조직 단위는 상하위 계층 구조로 관리
* 사용자는 1개의 조직에만 속하되, 역할은 다수 부여 가능
* 조직에 따라 시스템 내 자동 정책 설정 가능 (예: 조직장 승인 필요)
* `isActive = false`인 조직은 사용자/자산 매핑 불가

---

## \[6. 확장 고려 사항]

* 조직장 지정 필드(`orgManagerId`) 도입 가능
* 조직 단위 별 예산/결재 권한 한도 설정 가능
* 조직 별 자산 재고 비교 기능 구성 가능
