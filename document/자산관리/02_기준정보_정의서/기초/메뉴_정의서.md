- 내  용: 자산관리 시스템에서 메뉴(Menu)의 정의, 유형, 계층 구조, 역할 기반 접근 통제를 위한 기준 구조 정의서

---

## [1. 정의]

메뉴는 자산관리 시스템 내의 **화면(Screen), 기능(Function), URL 라우팅(Route)** 단위의  
기본 구성 요소로, 사용자에게 노출되는 기능 단위의 이름, 경로, 그룹을 의미한다.

메뉴는 **메뉴 코드(menuCode)**와 **URL 경로(path)**로 정의되며,  
조직 및 역할에 따라 **접근 권한 여부가 제어**된다.

---

## [2. 메뉴 구조 계층]

| 메뉴레벨 | 설명 |
|----------|------|
| 1레벨 | 대분류 (예: 자산관리, 수선관리, 결재관리) |
| 2레벨 | 중분류 (예: 자산등록, 자산조회, 대여처리) |
| 3레벨 | 세부 기능 (예: RFID 스캔 등록, 위치 이동 이력 등) |

---

## [3. 메뉴 속성 항목]

| 필드명 | 설명 |
|--------|------|
| menuCode | 메뉴 코드 (예: ASSET001, RENT001) |
| menuName | 사용자에게 표시되는 메뉴명 |
| path | 기능 라우팅 URL (예: /asset/register) |
| parentMenuId | 상위 메뉴 ID (계층 구조 설정) |
| menuLevel | 계층 깊이 (1, 2, 3 등) |
| displayOrder | 정렬 순서 |
| isActive | 사용 여부 (false인 경우 메뉴 숨김 처리) |

---

## [4. 메뉴와 역할의 관계]

- 메뉴 접근은 `ZCM_ROLE_MENU_MAP`을 통해 `ZCM_ROLE`과 연결됨
- 사용자는 `ZCM_USER → ZCM_USER_ROLE → ZCM_ROLE`을 통해 메뉴 접근 권한을 간접 보유
- `isActive = false` 이거나 권한이 없는 메뉴는 UI에서 비노출됨

---

## [5. 메뉴 코드 구성 규칙]

| 접두어 | 설명 |
|--------|------|
| ASSET | 자산 등록/조회 관련 |
| RENT | 대여 관련 |
| REPAIR | 수선 관련 |
| REFORM | 리폼 관련 |
| APPRO | 결재 관련 |
| ADMIN | 관리자 설정 관련 |

메뉴코드는 반드시 고유해야 하며, 메뉴 테이블의 `menuCode` 기준으로 일관성 유지

---

## [6. 메뉴 사용 예시]

| menuCode | menuName | path | level |
|----------|----------|------|-------|
| ASSET | 자산관리 | /asset | 1 |
| ASSET001 | 자산등록 | /asset/register | 2 |
| ASSET002 | 자산조회 | /asset/list | 2 |
| RENT001 | 대여등록 | /asset/rent | 2 |
| REPAIR001 | 수선접수 | /asset/repair/request | 2 |

---

## [7. 관리 기준]

- 메뉴 등록은 관리자만 가능
- 메뉴 비활성화 시 모든 사용자에게 미노출
- 메뉴는 기능 단위로 정의되며, 화면 단위 또는 API 호출 단위로 확장 가능

---

## [8. 확장 고려 사항]

- 메뉴별 기능 권한 세분화 (읽기, 쓰기, 승인 등)
- 모바일/웹 전용 메뉴 구분
- 메뉴별 접근 이력(`ZCM_MENU_LOG`) 테이블 설계 가능
