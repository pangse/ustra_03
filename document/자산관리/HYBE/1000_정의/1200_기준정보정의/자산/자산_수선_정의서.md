# 문서명: 자산수선_정의서.md
- 작성일: 2025-05-23
- 작성자: 송영근
- 내  용: 자산관리 시스템 내 자산의 수선, 점검, 수리 요청 및 결과 처리 흐름을 정의하고 상태 변화 및 연계 기준을 포함한 업무 정의 문서

---

## [1. 정의]

자산 수선은 자산이 손상되었거나 이상 상태로 판단되었을 때,  
**내부 혹은 외부 수리조치를 통해 자산을 복구하거나 점검하는 전체 행위**를 의미한다.  
수선은 정기점검, 고장수리, 리폼 준비 등의 흐름으로 포함될 수 있으며,  
자산의 상태는 수선 요청부터 완료까지 단계적으로 변경된다.

---

## [2. 업무 흐름]

```txt
[수선 요청] → [수선 진행] → [완료 처리] → [자산 상태 복귀]
                            ↳ [리폼 분기 가능]
````

* 수선 중 자산은 `REPAIR` 상태로 간주되며
* 완료 후에는 `ACTIVE`, 또는 `REFORMED` 상태로 전환 가능

---

## \[3. 수선 상태 코드 (`REPAIR_STATUS`)]

| 코드         | 설명                      |
| ---------- | ----------------------- |
| REQUESTED  | 수선 요청됨                  |
| IN\_REPAIR | 수선 진행 중                 |
| COMPLETED  | 수선 완료됨                  |
| FAILED     | 수선 실패/중단됨               |
| REFORM     | 리폼 분기 대상 (수선 후 리폼으로 전환) |

---

## \[4. 수선 유형 코드 (`REPAIR_TYPE`)]

| 코드                 | 설명          |
| ------------------ | ----------- |
| DAMAGE\_FIX        | 물리적 파손 수리   |
| CLEANING           | 의류/장비 세탁/청소 |
| FUNCTION\_TEST     | 기능 점검       |
| DESIGN\_REPAIR     | 외형 보수       |
| INTERNAL\_REPAIR   | 내부팀 처리      |
| OUTSOURCED\_REPAIR | 외부업체 위탁 수리  |

---

## \[5. 연계 테이블 및 항목]

| 항목                          | 설명                                            |
| --------------------------- | --------------------------------------------- |
| 자산 마스터 (`ZFI_ASSET_MASTER`) | 자산 상태 갱신 필요: `REPAIR`, `ACTIVE`, `REFORMED` 등 |
| 사용자 (`ZCM_USER`)            | 요청자, 수리자                                      |
| 조직 (`ZCM_ORG_UNIT`)         | 요청 조직 기준                                      |
| 수선 이력 (`ZFI_ASSET_REPAIR`)  | 요청부터 완료까지의 정보 저장                              |
| 리폼 이력 (`ZFI_ASSET_REFORM`)  | 수선이 리폼으로 전환될 경우 연계                            |

---

## \[6. 처리 기준]

* 수선 중에는 해당 자산의 이동, 대여, 폐기 등이 제한됨
* 수선 완료 후에는 수리 내역, 처리일자, 수선자, 수선비용이 기록되어야 함
* 수선 실패 시 실패 사유를 반드시 기록하고 상태를 유지하거나 회수로 전환해야 함

---

## \[7. 확장 고려 사항]

* 외부 수리업체 정보(`vendor`)와 연계
* 수선비용 기록 및 SAP 전표 처리 가능성
* 수선 요청 시 첨부파일 필수화 가능 (사진, 요청서 등)
* 정기 점검 대상 자동 지정 기능 (사용 기간 기준)
